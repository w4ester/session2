### Slide 3: Name & Description (5:30 - 7:30)

**SCRIPT:**

"First thing - we need to name this GPT. I'm going to type in the Name field."

[Type slowly while speaking]

### Slide 4: Instructions (7:30 - 17:30) [THIS IS THE LONGEST SECTION]

**SCRIPT:**

"Okay, this is the most important part. This is where we give the GPT its personality, its rules, its brain. I'm going to type this slowly. You can either copy-paste from the workshop site, OR type along with me. Your choice."

[Pull up the instructions template on screen]

"I'm starting in the Instructions field. Here we go:"

[Type slowly, section by section. Read each line out loud as you type.]

"You... are... an... Executive Communication Assistant... for... [Your Company]."

[Pause]

"Again, use your real company name."

[Continue typing]

"Your Role:
- Draft board presentations, investor updates, and strategic briefs
- Always use a confident, clear, executive tone
- Include data-driven insights and metrics
- Structure content for busy executives (top-line summary first)
- Ask clarifying questions before drafting"

[Pause between each bullet. Look up at the audience.]

"Everyone still with me? This is telling the GPT how to behave. What tone to use. What structure to follow."

[Continue]

"Tone & Style:
- Professional but not stuffy
- Data-driven and factual
- Strategic focus (big picture + key details)
- Concise - every word counts
- Action-oriented"

[Pause]

"This is YOUR voice. If you want a different tone - more formal, less formal, more technical - change these words. Make it match how YOU actually write."

[Continue with Content Structure section]

"Content Structure:
1. Executive Summary (3-5 bullet points)
2. Situation/Context
3. Data & Insights
4. Recommendations
5. Next Steps"

[Pause]

"This is the template it will follow. Every time. Consistent structure = easier for your audience to consume."

[Continue with "Before drafting" section]

"Before drafting, always ask:
- What is the audience? (Board, investors, internal team?)
- What is the key message?
- What decisions need to be made?
- What data should be highlighted?"

[Pause]

"This is critical. We want the GPT to ask US questions, not make assumptions. That way, the output is exactly what we need."

[Final section - "Never"]

"Never:
- Use jargon without explanation
- Make recommendations without data
- Write long paragraphs (use bullets)
- Assume context - ask questions"

[Pause, look up]

"These are hard rules. Things the GPT should NEVER do. This prevents it from producing generic corporate jargon nonsense."

[Wait 10 seconds]

"Okay. Look at your Instructions field. It should have all of that. Take 30 seconds. Read through it. Make any small tweaks you want. This is YOUR GPT, not mine."

[WAIT 30 SECONDS - this is important for them to personalize]

"Great. Everyone ready? Next step."

### Slide 6: Enable Capabilities (25:30 - 30:30)

**SCRIPT:**

"Next: Capabilities. These are the tools we give the GPT."

[Check the box on screen]

"DALL-E: This generates images. Simple diagrams, charts, visual explanations. If you want the GPT to create a diagram of your 3-year strategy, this does it. I'm enabling it too."

[Check the box]

"Code Interpreter: This is for analyzing data, running calculations, creating charts from CSVs. For executive communication, you usually don't need this. You can always enable it later if you find you need it."

[Leave it unchecked]

"So I've got Web Browsing and DALL-E checked. Code Interpreter unchecked. You can do the same, or make a different choice. Totally up to you and what you need."

## üö® Backup Plans (If Things Go Wrong)

### If GPT Editor is Down:
1. **Acknowledge immediately:** "Looks like OpenAI's servers are having issues. This happens."
2. **Show pre-built GPT:** "I'm going to show you a GPT I built earlier. Watch how it works."
3. **Walk through instructions:** Use slides to explain each section verbally
4. **Provide written guide:** "Everything is in the workshop site. You can build this later."
5. **Pivot to strategy:** Spend more time on the Delegation Playbook and ROI discussion

### If Someone Can't Access ChatGPT Plus:
1. **Pair them up:** "Share a screen with someone next to you"
2. **Take notes:** "Follow along, take notes, build it after you upgrade"
3. **Show value:** Focus on the ROI and strategic value, not just the technical build
4. **Offer help after:** "Come find me after and I'll help you get set up"

### If You Fall Behind Schedule:
**After 20 minutes, if still on Slide 4:**
- Speed up typing (copy-paste more)
- Cut personalization time
- Say "We're running behind, going to move faster"

**After 30 minutes, if still on Slide 6:**
- Skip detailed testing (show your own test only)
- Skip Q&A entirely
- End exactly at 40 minutes with "All details are on the workshop site"

**After 35 minutes, if still on Slide 7:**
- Skip Delegation Playbook (Slide 9)
- Do quick Save & Share
- Direct them to resources: "Everything else is on the workshop site"

### If You're Ahead of Schedule:
**At 30 minutes, if on Slide 8:**
- Do more extensive testing (try all 3 test prompts)
- Show advanced features (web browsing live demo)
- Take audience questions during the build

**At 35 minutes, if on Slide 9:**
- Spend more time on Delegation Playbook
- Do deeper dive into ROI calculations
- Share real case studies and success stories
- Take Q&A questions

## üéì Key Success Factors

### What Makes This Workshop Work:

1. **Live Build Approach:** Everyone builds together in real-time
   - Benefit: Immediate hands-on experience
   - Risk: Technical issues can derail entire session
   - Mitigation: Have backup plans ready

2. **CEO-Specific Content:** Executive Communication Assistant
   - Benefit: Directly relevant to their daily work
   - Risk: Some CEOs may not write their own comms
   - Mitigation: Position as "understanding what's possible to delegate"

3. **Tight Timing:** 40 minutes forces focus
   - Benefit: Respects executive time, high engagement
   - Risk: May feel rushed
   - Mitigation: Provide written guide for post-workshop reference

4. **Delegation Playbook:** Shows how to scale
   - Benefit: Makes it about organizational transformation, not just personal productivity
   - Risk: May be too much information at the end
   - Mitigation: Keep it high-level, detailed version on website

### Risks and Benefits of Live Build Approach:

**Risks:**
- Technical failures (internet, platform downtime)
- Participants without ChatGPT Plus
- Varying technical skill levels
- Time pressure creates stress
- Hard to help individuals without slowing group

**Benefits:**
- Immediate hands-on learning
- Everyone leaves with working tool
- See instant value/results
- Forces engagement (can't be passive)
- Builds confidence through doing
- Creates "we did this together" shared experience

**How to Maximize Benefits:**
- Practice extensively beforehand
- Have backup plans for every failure mode
- Provide written guide for reference
- Set clear expectations upfront
- Celebrate small wins throughout
- Offer post-workshop support
